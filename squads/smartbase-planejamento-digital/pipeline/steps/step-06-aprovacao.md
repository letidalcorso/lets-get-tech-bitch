# Step 06 — Aprovação Final do Planejamento

**Tipo:** Checkpoint  
**Objetivo:** Apresentar o planejamento completo para aprovação antes de compilar o documento final.

---

## Fluxo

### 1. Apresentar o resumo de todos os entregáveis

Exibir ao usuário:
- Diagnóstico e posicionamento (Diana Digital)
- Highlights da estratégia Instagram: pilares e mix de formatos
- Highlights da estratégia SEO: top 5 palavras-chave, top 3 pautas
- Visão geral do calendário: mês 1, 2 e 3

### 2. Solicitar aprovação via AskUserQuestion

**Pergunta:** Como está o planejamento?
- Aprovado — compilar o documento final
- Ajustar estratégia Instagram — voltar para o Igor com feedback
- Ajustar estratégia SEO — voltar para a Sofia com feedback
- Ajustar calendário editorial — voltar para o Lucas com feedback
- Refazer diagnóstico — voltar para a Diana com feedback

### 3. Se aprovado

Acionar o Compilador Final (Step 07) para consolidar tudo em um único documento.
Registrar em memories.md: data do planejamento, horizonte, principais decisões estratégicas.

### 4. Se ajuste solicitado

Coletar feedback específico do usuário e re-executar o step correspondente com o feedback como input adicional.

---

## Instruções para o checkpoint

- Sempre mostrar pelo menos a tabela de visão geral do calendário neste checkpoint
- Não re-executar nenhum agente sem coletar feedback específico do usuário
- Registrar aprendizados em memories.md após aprovação
- Perguntar se o usuário quer o documento final compilado (Step 07) ou prefere os arquivos separados

## Output esperado

- Planejamento aprovado pelo usuário
- Todos os arquivos individuais em `output/{run-folder}/`
